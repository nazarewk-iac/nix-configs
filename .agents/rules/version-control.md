# Version Control

See [jujutsu-vcs.md](jujutsu-vcs.md) for the full jj mandate and patterns this repo uses.

- Always leave an empty `jj` change on top when finishing work.
- Never push changes — the user reviews and pushes.
- Use conventional commit format (`feat:`, `docs:`, `chore:`, `fix:`).
