# MCP Setup

> **Agent summary:** [.agents/rules/mcp-setup.md](../.agents/rules/mcp-setup.md)

This repo uses [mcp-gateway](https://github.com/google-deepmind/mcp-gateway) to aggregate
multiple MCP servers behind a single stdio interface registered with Claude Code. The entire
setup is managed through the `kdn.mcp` devenv slot in `modules/slots/mcp/`.

---

## Architecture

```
Claude Code
    │
    │  stdio (one connection, ~14 meta-tools)
    ▼
mcp-gateway                    ← aggregates all backends
    ├── git                     (mcp-servers-nix)
    ├── filesystem              (mcp-servers-nix)
    ├── nixos                   (mcp-servers-nix)
    ├── sequential-thinking     (mcp-servers-nix)
    ├── time                    (mcp-servers-nix)
    ├── fetch                   (mcp-servers-nix)
    ├── devenv                  (extraBackend: "devenv mcp")
    ├── jj                      (extraBackend: kdn.jj-mcp)
    ├── memory-public           (basic-memory public KB)
    └── memory-sensitive        (basic-memory sensitive KB)
```

Claude Code sees **one MCP server** (`mcp-gateway`) exposing ~14 meta-tools regardless of how
many backends are configured. The gateway multiplexes calls to the appropriate backend
transparently.

---

## How it's wired up

The devenv slot (`modules/slots/mcp/default.nix`) does three things on each build:

1. **Evaluates** `kdn.mcp.programs` through `mcp-servers-nix` to resolve server commands into
   backend stanzas
2. **Generates** `mcp-gateway.yaml` in the Nix store and symlinks it to
   `.devenv/mcp-gateway.yaml` on `enterShell`
3. **Registers** `mcp-gateway` with Claude Code as a stdio server via `claude.code.mcpServers`

The registered command is a small wrapper script:

```bash
exec mcp-gateway serve --stdio -c "${DEVENV_ROOT}/.devenv/mcp-gateway.yaml"
```

The `$DEVENV_ROOT` expansion happens at runtime, so `.mcp.json` (generated by devenv from
`claude.code.mcpServers`) is a static file that never changes between builds — only the
symlink target at `.devenv/mcp-gateway.yaml` changes.

## Configuration

Configured in `devenv.nix`:

```nix
kdn.mcp = {
  enable = true;

  # mcp-servers-nix program declarations (see github.com/natsukium/mcp-servers-nix)
  programs = {
    git.enable = true;
    nixos.enable = true;
    filesystem = { enable = true; args = [ "/nix/store" ]; };
    sequential-thinking.enable = true;
    time.enable = true;
    fetch.enable = true;
  };

  # Extra backends not covered by mcp-servers-nix
  extraBackends = {
    devenv = {
      command = "devenv mcp";
      description = "devenv — search nixpkgs packages and devenv options";
      env.DEVENV_ROOT = toString ./.;
    };
    jj = {
      command = "${pkgs.kdn.jj-mcp}/bin/jj-mcp";
      description = "jj — Jujutsu version control tools";
    };
  };

  # Optional: override gateway bind address/port (defaults: 127.0.0.1:39400)
  # host = "127.0.0.1";
  # port = 39400;
};
```

### Adding a backend

**Server covered by mcp-servers-nix:**
```nix
kdn.mcp.programs.github.enable = true;
```

**Custom stdio server:**
```nix
kdn.mcp.extraBackends.my-server = {
  command = "${pkgs.my-server}/bin/my-server --some-flag";
  description = "brief description shown in gateway meta-tools";
};
```

**HTTP / SSE server:**
```nix
kdn.mcp.extraBackends.my-http = {
  http_url = "http://localhost:8080/mcp";
  description = "...";
};
```

After adding a backend: rebuild devenv, re-enter the shell to update the symlink, then
reload the backend in Claude Code (see Lifecycle below).

## basic-memory knowledge bases

Enabled via `kdn.mcp.basic-memory.enable = true` (a submodule of the mcp slot). Adds two
backends:

| Backend | CLI alias | Purpose |
|---|---|---|
| `memory-public` | `bmp` | Open-source tooling, public Nix knowledge |
| `memory-sensitive` | `bms` | Private / confidential knowledge |

These also install an always-on Claude rule:
- `.claude/rules/basic-memory.md` — routing rules (public vs sensitive, OKF format)

Projects live under `~/.local/share/kdn-nix-configs/knowledge/{public,sensitive}/`.

## Lifecycle

| Event | What happens |
|---|---|
| `devenv shell` | Symlinks `.devenv/mcp-gateway.yaml` → generated config in `/nix/store/` |
| Claude Code starts | Launches wrapper → starts `mcp-gateway` process connected to backends |
| Backend config changes | Rebuild devenv, re-enter shell; reload in Claude session (see below) |
| New backend added | Same as above; `.mcp.json` unchanged since only the YAML symlink moves |

**Reloading changed backends without restarting the session:**
```
# via mcp-gateway meta-tools inside Claude Code:
gateway_kill_server <name>
gateway_revive_server <name>
```

## Troubleshooting

**Inspect the generated config:**
```bash
cat .devenv/mcp-gateway.yaml
ls -la .devenv/mcp-gateway.yaml   # should be symlink into /nix/store/
```

**Run the gateway directly to see startup errors:**
```bash
mcp-gateway serve -c .devenv/mcp-gateway.yaml
```

**List registered servers and tools (inside Claude Code session):**
Use the `gateway_list_servers` and `gateway_list_tools` meta-tools.

**To debug JSON-RPC traffic interactively:**
See [mcpsnoop.md](mcpsnoop.md). mcpsnoop is enabled by default — just run `mcpsnoop` in a
second terminal while Claude Code is running.
